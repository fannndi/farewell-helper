🏛️ 4 Pillars

OpenCode

Agent Runtime
  • 2 agents: build (full access) + plan (read-only)
  • PLAN enforced via permission.edit: deny + permission.bash: deny
  • Instructions: PERSONA.md + PROTOCOL.md
  • MCP server: py -m farewell_helper.mcp --stdio (4 tools)
  • 6 slash commands: /daily /sync /verify /start /health /pre-commit

9Router

Model Gateway at :20128
  • Auth via NINEROUTER_API_KEY env var
  • 4 combo tiers: FREE_Model, Pro_Plan, Execution_Paid, Execution_Free
  • Simple HTTP client: chat(), models(), ping()
  • Source of truth: farewell.combos.jsonc → resolved by sync.py
  • Fallback strategy for cache affinity; Round-robin for diversity

Skills

Engineering Method + Domain Knowledge
  • 4 skill files under skills/
  • Trigger: frontmatter description with "Use when..." clause
  • farewell-persona: identity, YAGNI, push-back
  • farewell-engineering: TDD, bug fixing, code review
  • farewell-flows: PLAN/BUILD, handoff, memory, DoD
  • farewell-9router: combo strategies, model profiles

Farewell Persona

AI Identity + Boss Identity + Protocol
  • PERSONA.md: Who AI is, who Boss is, values, YAGNI
  • PROTOCOL.md: Work modes, precision standard, rules, DoD
  • Memory: MEMORY.md (2.2K chars) + USER.md (1.4K chars) per project
  • Auto-glossary: AUTO-GLOSSARY.md per project
  • Session lineage via lineage.json per project

🔄 Key Data Flows

🚀 Session Start

  1. OpenCode loads PERSONA.md + PROTOCOL.md as instructions
  2. /start verifies persona files exist
  3. Pings 9Router health endpoint
  4. Shows active project context
  5. AI reads persona → knows who they are + who Boss is

📅 Daily Check

  1. /daily: verify persona
  2. sync: read farewell.combos.jsonc
  3. Resolve ${PLACEHOLDER}s in opencode.template.jsonc
  4. Write opencode.jsonc
  5. Ping 9Router → report health

📝 Handoff Cycle

  1. done: pre-commit check → git add → git commit → git push
  2. generate_handoff(): writes .farewell/memory/{code}-{name}/handoff-*.md
  3. Updates MEMORY.md with session summary
  4. Archives TODO.md, creates fresh task list
  5. Next session: last_handoff() resumes from last handoff

🏗️ Multi-Repo

  1. setup-project /path/to/repo: detects archetype, injects .farewell/
  2. Registers in .farewell/projects.txt
  3. project switch <code>: switches active project context
  4. Each project has isolated MEMORY.md, USER.md, AUTO-GLOSSARY.md
  5. Zero footprint on target repo (only .farewell/ dir)
